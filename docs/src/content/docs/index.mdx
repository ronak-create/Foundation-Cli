## Why Foundation CLI?

<CardGrid stagger>
  <Card title="Stack Flexibility" icon="open-book">
    Choose from 4 frontends, 4 backends, 5 databases, 5 auth providers, and more. Any combination
    works together.
  </Card>
  <Card title="Conflict Detection" icon="warning">
    The DAG-based resolver automatically detects incompatible modules before they cause issues.
  </Card>
  <Card title="Config Merging" icon="setting">
    Deep merge for package.json, tsconfig.json, .env, docker-compose — no more manual fixing.
  </Card>
  <Card title="Atomic Writes" icon="rocket">
    Files are staged to temp dir and committed atomically. On failure, nothing reaches your disk.
  </Card>
</CardGrid>

## Quick Example

```bash
# One-shot, no install needed
npx @systemlabs/foundation-cli create my-saas

# Or install globally
npm install -g @systemlabs/foundation-cli
foundation create my-saas --preset saas
```
